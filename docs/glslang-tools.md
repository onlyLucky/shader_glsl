# glslang 工具使用指南

## 概述

glslang 是 Khronos 官方的 GLSL/HLSL 编译器和验证器，主要用于将着色器代码编译为 SPIR-V 二进制格式。

## 基本语法

```bash
glslangValidator [选项] [文件]
```

## 支持的文件类型

| 扩展名 | 阶段 |
|---|---|
| `.vert` | 顶点着色器 (Vertex) |
| `.frag` | 片段着色器 (Fragment) |
| `.geom` | 几何着色器 (Geometry) |
| `.tesc` | 曲面细分控制 (Tessellation Control) |
| `.tese` | 曲面细分评估 (Tessellation Evaluation) |
| `.comp` | 计算着色器 (Compute) |
| `.mesh` | 网格着色器 (Mesh) |
| `.task` | 任务着色器 (Task) |

## 常用命令

### 1. 验证着色器语法

```bash
# 基本验证
glslangValidator shader.frag

# 指定目标环境
glslangValidator --target-env opengl shader.frag
glslangValidator --target-env vulkan shader.frag
```

### 2. 编译为 SPIR-V

```bash
# Vulkan 语义
glslangValidator -V shader.frag -o shader.spv

# OpenGL 语义
glslangValidator -G shader.frag -o shader.spv
```

### 3. 预处理输出

```bash
# 查看宏展开结果
glslangValidator -E shader.frag
```

### 4. 查看 SPIR-V 可读格式

```bash
# 输出人类可读的 SPIR-V
glslangValidator -H shader.frag
```

### 5. 定义预处理宏

```bash
# 定义单个宏
glslangValidator -DDEBUG shader.frag

# 定义带值的宏
glslangValidator -DVERSION=3 shader.frag

# 定义多个宏
glslangValidator -DDEBUG -DVERSION=3 shader.frag
```

### 6. 指定着色器阶段

```bash
# 当文件扩展名不明确时
glslangValidator -S frag myshader.txt
```

### 7. 添加 include 搜索路径

```bash
glslangValidator -I./include shader.frag
```

### 8. 优化选项

```bash
# 禁用优化
glslangValidator -Od shader.frag

# 优化体积
glslangValidator -Os shader.frag
```

## 常用目标环境

| 环境 | 说明 |
|---|---|
| `opengl` | OpenGL |
| `opengl100` | OpenGL 1.0 |
| `vulkan` | Vulkan |
| `vulkan100` | Vulkan 1.0 |
| `vulkan110` | Vulkan 1.1 |

## 版本要求

| GLSL 版本 | 说明 |
|---|---|
| GLSL ES 1.00 | 旧语法，使用 `gl_FragColor`，不支持 SPIR-V |
| GLSL ES 3.00 | 现代语法，需要显式 `out` 变量 |
| GLSL ES 3.10 | 支持 SPIR-V 编译的最低版本 |

## 实际示例

### 验证并修复着色器

```bash
# 验证旧语法（会报错）
glslangValidator --target-env opengl test.frag

# 错误信息：
# ERROR: #version: ES shaders for SPIR-V require version 310 or higher
# ERROR: 'gl_FragColor' : undeclared identifier

# 修复为现代语法后验证通过
glslangValidator --target-env opengl test_fixed.frag
```

### 编译 SPIR-V

```bash
# 编译
glslangValidator -V shader.frag -o shader.spv

# 查看文件大小
ls -la shader.spv
```

### 查看 SPIR-V 内容

```bash
# 查看可读格式
glslangValidator -H shader.frag

# 输出示例：
# // Module Version 10000
# // Generated by (magic number): 8000b
# Capability Shader
# EntryPoint Fragment 4 "main" 9
# Source ESSL 310
```

## VSCode 集成

在 `.vscode/settings.json` 中配置：

```json
{
  "glsl-lint.glslangValidatorPath": "/opt/homebrew/bin/glslangValidator",
  "glsl-lint.additionalOptions": "--target-env opengl"
}
```

## 常见问题

### 1. 版本不匹配

**错误**：`ES shaders for SPIR-V require version 310 or higher`

**解决**：将 `#version` 升级到 310 es 或更高

### 2. 未声明的输出变量

**错误**：`'gl_FragColor' : undeclared identifier`

**解决**：使用显式 `out` 变量替代内建 `gl_FragColor`

```glsl
// 旧语法
gl_FragColor = vec4(1.0);

// 新语法
out vec4 fragColor;
fragColor = vec4(1.0);
```

### 3. 缺少 location 声明

**错误**：`'location' : SPIR-V requires location for user input/output`

**解决**：为输出变量添加 location

```glsl
layout(location = 0) out vec4 fragColor;
```
